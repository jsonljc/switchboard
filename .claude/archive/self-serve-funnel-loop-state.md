## Slices (do ONE per loop iteration; each its own PR-sized branch off fresh origin/main)

1. ROOT REDIRECT: unauthenticated `/` -> `/welcome` (marketing); authed `/` -> app. Keep public paths public; keep the existing exact-`/` auth logic for authed users. Update `middleware.ts` + the matcher; test in `src/__tests__/middleware.test.ts`. (auth stop-glob -> surface.)
2. REGISTER PAGE: `app/register/page.tsx` (email+password) -> POST `/api/auth/register`. Mode-aware: when `isSelfServeSignupOpen()` is false, show the waitlist CTA/form instead of the form (or render the 403 path gracefully). On success -> the existing verify-email screen (`/login?verify=true`). Link from `/login` ("Create account") and gate the landing CTAs (slice 4). Static `NEXT_PUBLIC_LAUNCH_MODE` read only (F-20 rule: never bracket/dynamic env in client). Add page + wiring tests.
3. WAITLIST FORM: an email-capture component on `/welcome` (and shown by the register page when signup closed) -> POST `/api/waitlist`. Success/error/duplicate states. Component test.
4. CTA REPOINT: replace `mailto:` CTAs (pricing cards, hero, closer, dock, topbar "Get started") with `/register` when open OR the waitlist form when closed, driven by the launch-mode flag. Snapshot/wiring tests.
5. RESEND-VERIFICATION UI: a "Resend verification email" affordance on `/login?verify=true` (and/or an in-app banner for an authenticated-but-unverified user) -> POST `/api/dashboard/auth/resend-verification`. Test.
