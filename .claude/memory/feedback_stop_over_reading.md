---
name: Stop over-reading before writing
description: Don't spend excessive time reading reference files before producing output — gather enough context then write
type: feedback
---

When writing implementation plans, don't over-read reference files. Gather the essential context (spec + 2-3 key patterns) then start writing the plan immediately. The user got frustrated when 4+ rounds of file reads happened before any plan output was produced.

**Why:** Excessive reading wastes time and delays the actual deliverable. The user wants action, not preparation theater.

**How to apply:** For plan-writing tasks, read the spec and at most 1-2 key pattern files in the first round, then immediately start writing. Look up additional details inline as needed while writing, don't front-load all research.
