# `/activity` (Slice D3) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Mercury read-only `/activity` register over `AuditEntry` — the fourth Tools-tier surface, behind `NEXT_PUBLIC_ACTIVITY_LIVE`. Two-chip filter (`Operational | All events`), inline drawer, cursor pagination, no polling.

**Architecture:** Backend stays surface-agnostic — projection types extend the existing `packages/schemas/src/audit.ts`; a new `listAuditEntriesForBrowse` core function projects raw `AuditEntry` rows from a new `LedgerStorage.listForBrowse` storage method (with thin `AuditLedger.listForBrowse` pass-through) into a redacted `AuditEntryBrowseRow`. Cursor encoding and trimming live in core. The existing `/api/audit/*` Fastify routes, `useAudit()` hook, and `activity-translator.ts` are untouched and continue to feed live-signal-popover and event-history components. Frontend is the only place the word "activity" appears.

**Tech Stack:** TypeScript, pnpm + Turbo monorepo, Zod schemas, Prisma + Postgres, Fastify (API), Next.js App Router (dashboard), TanStack React Query, vitest, Tailwind + CSS modules.

**Spec:** `docs/superpowers/specs/2026-05-10-activity-d3-design.md` (decisions ledger §2.0).

**PR shape:** Two PRs targeting `main`.

- **PR-D3a** = Tasks 1a–7a (backend: schemas + core + db + api + Next proxy + Prisma index).
- **PR-D3b** = Tasks 1b–6b (frontend page + tests). Gated by `NEXT_PUBLIC_ACTIVITY_LIVE=false` until staging review.

After every task: `pnpm typecheck` and the relevant `pnpm --filter <pkg> test` must be green before commit. CI runs `pnpm lint` + `pnpm test` + `pnpm typecheck` on PR open; fix errors locally, never on the PR.

If `pnpm typecheck` reports missing exports from `@switchboard/schemas` / `@switchboard/db` / `@switchboard/core` after a schema or store edit, run `pnpm reset` (per `CLAUDE.md`) before debugging — the dist/ folders need refreshing across the layers.

## Implementer watch-outs

- **`OPERATIONAL_AUDIT_EVENT_TYPES` is a `readonly` tuple.** When passing to a Prisma `IN` clause use the spread (`[...OPERATIONAL_AUDIT_EVENT_TYPES]`) so Prisma treats it as a mutable array.
- **`base64url`, not `base64`,** for cursor encoding. The cursor goes in URL query strings.
- **Cursor decode is tolerant.** Return `null` on any parse failure; never throw. Operators may share URLs across staging/prod with ledger drift.
- **`(timestamp, id) DESC` tuple comparison.** Postgres doesn't support tuple `<` directly through Prisma's where clauses, so use the OR-decomposition shown in §5.3 of the spec: `{ timestamp: { lt: cursorDate } } OR { timestamp: cursorDate, id: { lt: cursor.id } }`.
- **Visibility filter is server-side**, in the WHERE clause, not in the projection. Admin/system entries never leave the database for the browse endpoint.
- **Snapshot redaction is value-based, not key-based.** The projection emits `snapshotKeys: string[]` — a list of allowlisted key _names_, never a `Record<string, unknown>`. Any future refactor that "helpfully" surfaces values trips the drawer regression test.
- **Existing `/api/audit/*` is parallel infrastructure.** Resist the urge to merge endpoints. They serve different surfaces with different shapes; convergence is a post-D5 cleanup.
- **No polling.** Set explicit `staleTime: 30_000` on the query (TanStack Query default is `0`); no `refetchInterval`. Polling on a paginated list breaks cursor stability.
- **Per `feedback_prisma_migrate_dev_tty.md`:** in agent sessions use `prisma migrate diff --from-url --to-schema-datamodel --script` then `prisma migrate deploy`. `migrate dev` blocks on TTY prompts.
- **Per `feedback_api_test_mocked_prisma.md`:** API tests use mocked Prisma (or an `InMemoryLedgerStorage` wrapped in `new AuditLedger(...)`), not real Postgres. CI has no Postgres.
- **Branch hygiene per `CLAUDE.md`:** verify `git branch --show-current` matches the worktree before each commit; pre-commit hook will warn if a docs/feat slug mismatches.
