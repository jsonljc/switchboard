# OpenClaw Session Runtime — Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the session persistence layer, role manifest system, and session API endpoints so that one role-scoped agent (ad-operator) can run end-to-end through the governed orchestrator path with approval pause/resume.

**Architecture:** Session state machine lives in `packages/core/src/sessions/` following the existing store-interface pattern. Prisma models and store implementations in `packages/db/`. Role manifests in top-level `agent-roles/`. Session API endpoints and worker in `apps/api/`. The session layer is additive — no semantic changes to the existing orchestrator, execution service, or approval flow.

**Tech Stack:** TypeScript, Prisma ORM, Fastify, BullMQ, Vitest, Zod

**Spec:** `docs/superpowers/specs/2026-03-21-openclaw-session-runtime-design.md`

**Parallelism:** Tasks 1 (schemas), 2 (state machine), 4 (Prisma models), and 10 (role manifest) are independent and can be executed concurrently. Task 3 depends on 2. Task 5 depends on 3+4. Tasks 6-8 depend on 1+3. Task 9 depends on 6-8. Tasks 11-15 depend on 9+10. Tasks 16-20 are verification/integration and run after their subjects are complete.

Create `packages/core/src/sessions/state-machine.ts` with pure functions:

```typescript
import type { SessionStatus } from "@switchboard/schemas";

const TERMINAL_STATES: ReadonlySet<SessionStatus> = new Set(["completed", "failed", "cancelled"]);

const VALID_TRANSITIONS: ReadonlyMap<SessionStatus, ReadonlySet<SessionStatus>> = new Map([
  ["running", new Set(["paused", "completed", "failed", "cancelled"])],
  ["paused", new Set(["running", "cancelled"])], // paused → failed goes through cancellation, not direct failure
]);

export function isTerminal(status: SessionStatus): boolean {
  return TERMINAL_STATES.has(status);
}

export function canTransition(from: SessionStatus, to: SessionStatus): boolean {
  const allowed = VALID_TRANSITIONS.get(from);
  return allowed?.has(to) ?? false;
}

export type TransitionResult = { ok: true } | { ok: false; reason: string };

export function validateTransition(from: SessionStatus, to: SessionStatus): TransitionResult {
  if (isTerminal(from)) {
    return { ok: false, reason: `Cannot transition from terminal state '${from}'` };
  }
  if (!canTransition(from, to)) {
    return { ok: false, reason: `Invalid transition: '${from}' → '${to}'` };
  }
  return { ok: true };
}
```

import type { SessionStore, RunStore, PauseStore, ToolEventStore } from "./store-interfaces.js";
import type { AgentSession, AgentRun, AgentPause } from "./types.js";
import type {
  SessionStatus,
  PauseReason,
  AgentCheckpoint,
  ApprovalOutcome,
  ToolEvent,
  ResolvedRoleConfig,
  SessionResumePayload,
} from "@switchboard/schemas";
import { validateTransition, isTerminal } from "./state-machine.js";
import { buildResumePayload } from "./resume-payload-builder.js";
import { validateCheckpoint } from "./checkpoint-validator.js";
import crypto from "node:crypto";

export interface SessionManagerDeps {
  sessions: SessionStore;
  runs: RunStore;
  pauses: PauseStore;
  toolEvents: ToolEventStore;
}

export class SessionManagerImpl {
  constructor(private deps: SessionManagerDeps) {}

  async createSession(params: {
    organizationId: string;
    agentRole: string;
    principalId: string;
    goal: string;
    channel: string;
    roleConfig: ResolvedRoleConfig;
  }): Promise<{ session: AgentSession; run: AgentRun }> {
    // Check concurrent limit
    const activeCount = await this.deps.sessions.countActive(
      params.organizationId,
      params.agentRole,
    );
    if (activeCount >= params.roleConfig.maxConcurrentSessions) {
      throw new ConcurrentSessionLimitError(params.agentRole, activeCount);
    }

    const now = new Date();
    const session: AgentSession = {
      id: crypto.randomUUID(),
      ...params,
      status: "running",
      currentStep: 0,
      checkpoint: null,
      checkpointAt: null,
      toolHistory: [],
      approvalOutcomes: [],
      traceId: crypto.randomUUID(),
      createdAt: now,
      updatedAt: now,
    };

    const run: AgentRun = {
      id: crypto.randomUUID(),
      sessionId: session.id,
      runIndex: 0,
      triggerType: "initial",
      resumeContext: null,
      outcome: null,
      stepRange: null,
      startedAt: now,
      completedAt: null,
    };

    await this.deps.sessions.save(session);
    await this.deps.runs.save(run);

    return { session, run };
  }

  // ... remaining methods follow same pattern:
  // validate state, update stores, return result
}
```

Implement all methods from the spec's `SessionManager` interface, plus `resumeAfterApproval` which encapsulates the full resume orchestration. Each method:

1. Loads current state from stores
2. Validates the transition via `validateTransition()`
3. Updates stores atomically where possible
4. Returns the result

`resumeAfterApproval` is the key addition beyond the spec interface — it encapsulates:

1. `markResumable(pauseId, approvalOutcome)` — atomic CAS on resume token
2. `validateTransition("paused", "running")` — session status transition
3. Create new `AgentRun` with `triggerType: "resume_approval"` and `runIndex = runs.length`
4. Update session status to `running`
5. Return `{ session, run }` for the caller to enqueue the invocation job

This keeps all state transition logic inside the manager and prevents routes from reaching into store internals.

  } catch (err) {
    // Log but don't fail the approval response — resume is best-effort from the approval caller's perspective
    app.log.error({ err, approvalId: id }, "Failed to enqueue session resume");
  }
}
```

`resumeAfterApproval(approvalId, outcome)` internally:

1. Looks up the pause by `approvalId`
2. Returns `null` if no linked pause found (not a session-linked approval)
3. Calls `markResumable` (atomic CAS on `resumeStatus`)
4. Transitions session `paused → running`
5. Creates new `AgentRun`
6. Returns `{ session, run, resumeToken }`
