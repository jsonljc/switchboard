```markdown
# Editorial conventions — shared design decisions for Switchboard editorial-register surfaces

> Derived from `docs/design-prompts/locked/` on 2026-05-13. This file is **derived, not edited**: re-derive when the lock refreshes; don't hand-edit individual rules. New conventions discovered mid-redesign come back here as their own focused PR to `main`.

**Not in scope:** Mercury Tools tier (`--mercury-*` tokens, Tools-tier surfaces already shipped). Marketing site (`--sw-*` tokens). Those have their own doctrine elsewhere.

## Register scope

Switchboard's design splits into two registers that are never mixed within a surface:

- **Editorial** — agent homes, /mission, /approvals, /reports, /activity, /contacts, marketing site. Source Serif 4 prose, JetBrains Mono folios, warm cream + ink + hairline tokens.
- **Mercury / Tools tier** — `/settings`, `/automations`, and the previous incarnations of /activity, /contacts, /reports. Uses `--mercury-*` tokens.

This doc is editorial-only.

## Register boundaries (the two surfaces migrating from Mercury to editorial)

Two wave-2 surfaces are crossing the register boundary:

- **/activity** — currently at `apps/dashboard/src/app/(auth)/(mercury)/activity/`. Locked design `docs/design-prompts/locked/switchboard/project/activity-v2/Activity.html` is editorial. The /activity worktree must consume editorial tokens, not `--mercury-*`. The route-group migration (moving the directory out of `(mercury)/`) is a wave-2 decision; this doc states the register intent only.
- **/contacts** — currently at `apps/dashboard/src/app/(auth)/(mercury)/contacts/`. Locked design `docs/design-prompts/locked/switchboard/project/agent-home-v3/Pipeline.html` shares the agent-home shell — the strongest possible signal that contacts is now editorial.

## Authoring rules

1. **Derive every rule from a locked-mockup citation.** Format: `locked/switchboard/project/<dir>/<file>:<line-range>`. The locked mockup is the source of truth; opinion isn't.
2. **When the locked mockups disagree across surfaces, adjudicate explicitly.** Name the disagreement in one sentence, pick a winner, give a one-line rationale. Never paper over disagreement.
3. **Prescriptive, not exhaustive.** Lock the decisions that vary across the six worktrees. Don't re-specify per-surface designs.
4. **Doc is derived, not edited.** New conventions come back here as a focused PR to `main`.
```
