## Honesty rules (read first)

1. **Render only `ReportDataV1` fields. Invent nothing.** No metric exists outside the schema below.
2. **Money is whole SGD dollars, NOT cents.** The backend emits dollars (`14720` = S$14,720; CPC `0.34` = 34¬¢). Reuse the existing `format.ts` `fmtSGD` (dollar formatter, SGD threshold rules). **Never** divide by 100. Never a bare `$`‚Äî always`S$`.
3. **`attribution` is Alex + Riley ONLY ‚Äî there is no `mira` field.** Render Mira as **"Not set up yet"** (the same honest trust signal Home uses). Never fabricate a Mira number.
4. **No row-level tap-through endpoint exists** ("show me the 23 consults"). The funnel/campaign/managed depth is already in the payload (no extra fetch). A headline‚Üílist drill-down is pending backend ‚Äî if hinted at all, make it a quiet, contextual "coming soon" affordance on the number itself; never a banner, and never name internal concepts like "the events store".
5. **Attribution caveat:** revenue is attributed **at the time a consult is booked, not collected** (weighted by service price, 30-day window). State this in the colophon; actual collections may differ.

## Mental model (do not break)

A non-technical medspa/clinic owner doing a ~2-minute, phone-first check-in (literacy ‚âà Instagram + Square + WhatsApp). Results is the ONE place they answer **"is my AI team worth it, and what's working?"** It is the validated home for _optional_ investigative depth ‚Äî but it must read as a **warm owner check-in, not a SaaS/BI console.** Every label names an OUTCOME ("booked consults", "what your ads earned"), never an internal concept ("conversion funnel stage 4"). Lead with the answer; let the owner _choose_ to dig.

## Domain ‚Äî the real payload (`ReportDataV1`, locked ‚Äî DO NOT INVENT FIELDS)

`GET /api/dashboard/reports?window=THIS WEEK|THIS MONTH|THIS QUARTER` ‚Üí

```ts
interface ReportDataV1 {
  label: "THIS WEEK" | "THIS MONTH" | "THIS QUARTER";
  period: string; // "MAY 1 ‚Äî MAY 26"
  dateFolio: string; // typographic period anchor
  pullquote: PullQuoteCopy; // ALWAYS populated (never null)
  attribution: AttributionData;
  funnel: FunnelRowData[]; // SIX stages ‚Äî see below
  funnelNarrative: FunnelNarrative; // ALWAYS populated
  campaigns: CampaignRow[];
  cost: CostBreakdown;
  costNarrative: string;
  managedComparison: ManagedComparisonData | null;
}

// `text` carries the glyph: "‚Üë 18%", "‚Üì 6%", "‚Äî"
type Delta = { kind: "pos" | "neg" | "flat"; text: string };

// LLM narrative; the numbers (`value`, `cost`) are interpolated from real fields
interface PullQuoteCopy {
  pre: string;
  value: string;
  mid: string;
  cost: string;
  post: string;
}

interface AttributionData {
  total: number; // SGD DOLLARS ‚Äî revenue attributed this period
  delta: Delta; // ALWAYS populated (non-null) for attribution
  riley: { value: number; caption: string }; // ad-driven leads converted (dollars)
  alex: { value: number; caption: string }; // reply / reactivation conversions (dollars)
  // NOTE: there is NO mira field.
}

interface FunnelRowData {
  stage: string;
  n: number;
  label: string;
  delta: Delta | null;
} // funnel deltas CAN be null

interface FunnelNarrative {
  marker: string;
  text: string;
} // marker e.g. "Riley ¬∑ May 22"

interface CampaignRow {
  name: string;
  spend: number;
  impressions: number;
  inlineLinkClicks: number;
  costPerInlineLinkClick: number; // dollars (e.g. 0.34)
  inlineLinkClickCtr: number; // fraction (0.0133)
  leads: number;
  revenue: number; // dollars
  cpl: number | null;
  clickToLeadRate: number | null;
  roas: number;
}

interface CostBreakdown {
  paid: number; // SGD dollars ‚Äî the SWITCHBOARD SUBSCRIPTION / PRODUCT cost (from the plan price). NOT ad spend.
  alt: number; // a market-rate estimate of an SDR + agency
  saving: number; // alt ‚àí paid
}

// managedComparison ‚Äî a managed-vs-unmanaged ("with us vs without") comparison, NOT a flat metrics block:
interface ManagedComparisonData {
  ads: ManagedComparisonPair | null; // independently nullable
  conversations: ManagedComparisonPair | null; // independently nullable
  source: "in-period-cohort" | "pre-switchboard-baseline";
  emptyMessage?: string;
}
interface ManagedComparisonPair {
  managed: Metrics;
  unmanaged: Metrics;
  delta: Delta;
}
interface Metrics {
  spend: number;
  revenue?: number;
  roas?: number;
  replies?: number;
  conversionRate?: number;
  replyMinutesP50?: number;
}
// Not every metric is populated (e.g. conversionRate / replyMinutesP50 may be absent) ‚Äî filter absent metrics, never fabricate.
```

**Derived values ‚Äî compute honestly:**

- **Booked consults** = the Bookings funnel stage. The funnel has **SIX** stages in fixed order ‚Äî `Impressions, Clicks, Landing page views, Leads, Bookings, Customers` ‚Äî so Bookings is **not** the last row. Get it by name: `funnel.find(r => r.stage === "Bookings")?.n ?? 0`. **Never** hardcode `funnel[4]`. Render each row's `stage` straight from the wire (the producer emits **"Landing page views"**, not "Landing visits").
- **Revenue** = `attribution.total`.
- **Real ad spend** = `Œ£ campaigns[].spend` ‚Äî **NOT `cost.paid`** (that is the subscription cost).
- **Per-campaign `roas` is the only first-class return ratio.** **Do NOT compute a hero "return ratio"** (`attribution.total / cost.paid` or `/ ad spend`): `attribution.total` folds in Alex's no-ad-cost reactivations, so any such ratio is a category error that misleads the owner. **Do NOT show an "avg S$/consult"** for the same reason (mixed-population revenue √∑ consults).

## Actions / endpoints (real)

- Window switch ‚Üí re-query `?window=`. Default **This month**.
- Recompute ‚Üí `POST /api/dashboard/reports/refresh?window=‚Ä¶` (invalidates the 1-hour cache). Quiet control: "Recompute (updated 47m ago)" ‚Äî wire the age to real cache state; never hardcode it.
- Live vs sample ‚Üí `NEXT_PUBLIC_REPORTS_LIVE`. **Default (unset) = fixture mode** (existing `fixtures.ts`). Live needs a Meta Ads connection + populated revenue; the API **503s** when report stores aren't wired ‚Äî treat as an error state, not zeros.
- **No-Meta state is derived from connection status, NOT from funnel zeros.** Read `useConnections()` and show the connect banner only in live mode when the Meta connection is missing / not connected (as `reports-page.tsx` already does). **Verify the canonical `serviceId`:** `reports-page.tsx` keys on `"meta-ads"` while the API resolver queries `"meta"` ‚Äî confirm which is correct before relying on it.

## Layout ‚Äî lean headline, progressive disclosure (phone-first, top ‚Üí bottom)

1. **Header** ‚Äî "Results" (Newsreader); period segmented control (This week / This month / This quarter, default **This month**); `dateFolio` in mono; quiet "Recompute (updated Xm ago)".
2. **Verdict line** ‚Äî compose `pullquote` into one warm serif sentence: `{pre}` **`{value}`** `{mid}` **`{cost}`** `{post}` (emphasize `value` + `cost` with weight + an amber hairline). The free-text `post` is the model's read ‚Äî **attribute it with a "‚Äî Riley" byline**; if `post` is empty, render the numbers sentence alone.
3. **Hero outcomes (lean & honest)** ‚Äî three numbers, no derived metrics:
   - **Booked revenue** = `attribution.total`, large mono, with the `delta` badge inline (pos = amber up-glyph; neg = down-glyph in ink; flat = em-dash ‚Äî **no red/green**).
   - **Consults booked** = the Bookings stage `n` (co-hero), with its delta.
   - **Ad spend** = `Œ£ campaigns[].spend`, quieter. _(No return ratio, no avg/consult.)_
4. **What's working** _(promoted ABOVE the disclosure ‚Äî the owner's "so what do I do?")_ ‚Äî one warm line from `funnelNarrative` (Riley's read: `marker` + `text`) plus the best/worst campaign as a sentence, derived from max/min `campaigns[].roas`. No new endpoint.
5. **Per-agent contribution** ‚Äî Riley card (`attribution.riley`, teal dot) + Alex card (`attribution.alex`, coral dot), hairline-bordered, mono. A third **Mira card reads "Not set up yet"** (violet dot, muted). Agent color is identity only ‚Äî never fills a button.
6. **"Worth it?" ‚Äî cost vs value** (the renewal answer) ‚Äî three cells: "You pay" `cost.paid` (the subscription) ¬∑ "A salesperson + agency would cost" `cost.alt` (labelled a **market-rate estimate**) ¬∑ "You saved" `cost.saving`. **Co-weight "You pay" and "You saved"** so it reads as an auditable comparison, not a triumphant pitch. Then `costNarrative` in serif italic.
7. **See the details** ‚Äî one explicit disclosure (`--canvas-3` rule) revealing, in order:
   - **Funnel** ‚Äî stages as custom SVG/grid bars (no chart lib): stage label ¬∑ proportional bar (`n / max`) ¬∑ `label` string (mono) ¬∑ delta chip. Then `funnelNarrative` as an editorial byline. **When no Meta connection, hide the funnel block entirely** ‚Äî never a wall of zero-bars.
   - **Campaigns** ‚Äî phone = card list (best‚Üíworst by revenue; minimal sort chrome); desktop = table (Campaign ¬∑ Spend ¬∑ Impr ¬∑ Clicks+CTR ¬∑ CPC ¬∑ Leads ¬∑ CPL ¬∑ Revenue ¬∑ ROAS ¬∑ click‚Üílead), sticky first column, sortable (default revenue desc), mono, footer totals, ROAS emphasis by amber depth (never RGB). Prefer plain words over jargon.
   - **Managed comparison** ‚Äî render ONLY when `managedComparison !== null`: a **with-us-vs-without** before/after ‚Äî for each non-null Pair (`ads`, `conversations`), show `managed` vs `unmanaged` per metric + the `delta`, with a caption driven by `source` ("Compared to your pre-Switchboard baseline" / "‚Ä¶similar accounts this period"). Reuse the existing `managed-comparison.tsx` logic (partial population + `emptyMessage`). **Frame it as your own before/after ‚Äî never "if you had us run this for you".**
8. **Colophon** ‚Äî period restated, generation timestamp, **"Live data" / "Sample data"** badge (from `NEXT_PUBLIC_REPORTS_LIVE`), and the **attribution caveat** (booked-not-collected, weighted by service price) in `--ink-4`.

## State coverage

- **Loading:** skeleton blocks (hero + cards), no spinner.
- **Error / stores unavailable (503):** inline banner; the page shell stays ‚Äî never a full-page replace. Wire any cache-age copy to real state.
- **No Meta Ads connection** (live, from `useConnections`): calm banner above the hero ‚Äî revenue from Alex's reactivations still shows; campaigns + the funnel block are **hidden** (not zero-filled).
- **New org / no data yet:** a calm first-run line ‚Äî "Your first results land here once your team books a consult." `pullquote`/`funnelNarrative` are still populated by the backend, so do NOT use their nullness to detect first-run.
- **Negative `delta`:** muted down-glyph in ink, never red.
- **Mira:** "Not set up yet", never fabricated.

## Anti-patterns (avoid ‚Äî generic SaaS analytics is the default failure mode)

- **No invented fields / no fabricated Mira data / no fake tap-through lists.** Render only `ReportDataV1`.
- **No cents.** Whole SGD dollars via the existing `fmtSGD`. No bare `$`.
- **No `cost.paid` mislabeled as ad spend.** Real ad spend = `Œ£ campaigns[].spend`.
- **No hero return ratio and no avg/consult** ‚Äî they fold Alex's no-cost reactivations into ad-return math (category errors). Per-campaign `roas` is the only return ratio.
- **No `funnel[4]` indexing** ‚Äî the funnel is 6 stages; find Bookings by name.
- **No flattened managed-comparison** ‚Äî it's a managed-vs-unmanaged Pair with a `source`.
- No red/yellow/green. Delta + ROAS depth = glyph + amber depth only.
- No chart library; funnel/ROAS bars are custom SVG/grid.
- No sparkline-on-every-number, no trending widgets, no KPI-tile wall above the fold.
- Agent identity color NEVER fills a button or an action.
- Don't surface managed-comparison when null; don't show an empty placeholder.
- This is a warm owner check-in with optional depth ‚Äî not an operator/BI console.
