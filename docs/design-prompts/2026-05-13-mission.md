---

Design /mission — the operator's cross-agent command center for "Switchboard" (Singapore medspa operator dashboard). Agent home pages (/alex, /riley) cover "what is this one agent doing"; Mission Control answers "what needs me, right now, across everything." Spec context lives at docs/superpowers/specs/2026-04-30-console-as-home-dashboard-design.md. Greenfield design — no existing /mission route, but the data hooks all exist.

## Doctrine (non-negotiable)

Task-centric hierarchy: Queue → Agents → Activity. Answers:

1. What needs my attention? (action)
2. What can I do about it? (capability)
3. What context do I need to decide? (signal)
   This is NOT a metrics dashboard. Numbers are SUPPORTING context for the queue, not the headline. If you find yourself centering charts, you've gone wrong.

## Data sources (real hooks; verified in apps/dashboard/src/hooks)

useDashboardOverview() → DashboardOverview {
greeting: { period: "morning"|"afternoon"|"evening", operatorName },
stats: { pendingApprovals, newInquiriesToday, newInquiriesYesterday,
qualifiedLeads, bookingsToday, revenue7d (SGD cents),
openTasks, overdueTasks },
approvals: PendingApproval[] // top 3, links to /approvals/[id]
bookings: Booking[] // { id, startsAt, service, contactName,
// status, channel: whatsapp|telegram|dashboard }
funnel: { inquiry, qualified, booked, purchased, completed }
revenue: { total (SGD cents), count, topSource, periodDays }
tasks: OwnerTask[] // { id, title, dueAt, isOverdue, status }
activity: ActivityEntry[] // recent audit, with reasoning
}

useAgentRoster() → AgentRosterEntry[] — displayName, agentRole, status, tier ("alex" | "riley" | "mira" today)
useAgentState() → AgentStateEntry[] — activityStatus, currentTask, lastActionAt, lastActionSummary, metrics (links by agentRosterId)
useDecisionFeed() → cross-agent Decision[] discriminated union (approval | handoff) + counts
useRecommendations({ surface: "queue" }) → RecommendationApiRow[] (drafted agent actions awaiting operator confirmation)
useAudit() → AuditEntryResponse[] for the right-rail activity strip

Operator actions reachable from this surface:

- Approve / reject / patch a PendingApproval inline
- Dismiss or accept a Recommendation
- Undo a terminal recommendation (within undo window)
- Drill to /alex, /riley, /mira agent home
- Emergency halt an agent → POST /api/agents/[deploymentId]/halt
- Open /approvals, /activity, /reports

## Layout — three zones

Zone A (top, full width): "Now"

- Greeting: "Good morning, Jason" in display font (Instrument Sans) — small, not heroic
- One-line org pulse: "3 approvals waiting · 2 overdue tasks · 7 new inquiries today (+2 vs yesterday)" — mono numerics, hairline separators
- Right-aligned: emergency halt menu (per-agent, requires confirmation)

Zone B (middle, two-column 65/35): "Queue + Agents"
LEFT (65%): The Queue — chronological-by-urgency - Each item is either a PendingApproval, an OwnerTask, or a Recommendation, rendered as a uniform card so the operator scans one list, not three. - Card content: type chip ("APPROVAL" / "TASK" / "REC"), summary, meta (agent attribution + expiry/due/created), inline actions appropriate to type. Inline approve/reject for approvals (with binding-hash confirmation as required by approvals contract — collapse to a 1-tap "review & confirm" that opens an inline drawer, not a modal). - Empty state: "Inbox zero" with last-cleared timestamp.
RIGHT (35%): Agent strip — one card per agent in roster - DisplayName (Georgia italic for warmth; falls back to Instrument Sans) - activityStatus pill (idle / working / waiting / halted) — pill is hairline outline, no fill, no color - currentTask line (1 line, truncated) - lastActionSummary with relative time - Tap → /[agentKey] agent home

Zone C (bottom, full width): "Context strip"

- Three small cells side-by-side:
  1. Funnel mini (inquiry → qualified → booked → purchased → completed), horizontal bars, counts in mono. No chart library.
  2. Revenue 7d in SGD with topSource caption. Mono.
  3. Bookings today (next 3 starts, with channel glyph "wa"/"tg"/"dash")
- Below: micro activity tail — last 8 audit rows in a compact, low-contrast list. Tap → /activity (full feed).

## Design system

Use editorial register tokens — these are the values verified across globals.css today; if your codebase has both editorial (warm cream + coral) and operator (warm off-white + amber), prefer the operator-surface values:

- --sw-base #F5F3F0, --sw-surface #EDEAE5, --sw-border #DDD9D3
- --sw-text-primary #1A1714, --sw-text-muted #9C958F
- --sw-accent #A07850 (warm amber) for primary CTAs only
- Display font: Instrument Sans (sparingly, greeting + zone titles)
- Body: Inter; Mono: JetBrains Mono for all numerics
- Italic Georgia is acceptable for agent display names in the agent strip

Max content width 80rem; page-width 74rem outside.

## Anti-patterns

- No giant KPI tiles. Pulse line is enough.
- No charts. Mini funnel is just bars; no recharts/d3.
- No traffic-light status colors. Pills are hairline outlines.
- No icons. Use text glyphs ("wa", "tg", "USR", "AGT") in mono.
- No "real-time" mode by default — manual refresh, "fetched Xm ago" caption.
- Don't center revenue or bookings. The queue is the headline.

## Deliverable

Single React + Tailwind page with three zones, queue card component, agent strip card, mini funnel SVG. Realistic fixtures: 9 queue items (4 approvals spanning risk levels, 3 owner tasks 1 overdue, 2 recommendations), 3 agents (alex working, riley idle, mira waiting-on-approval), pulse numbers, a non-trivial funnel (200 / 80 / 35 / 24 / 18), revenue S$14,720 7d, 4 bookings today across all 3 channels. Include the emergency-halt confirmation pattern as a small inline component.
