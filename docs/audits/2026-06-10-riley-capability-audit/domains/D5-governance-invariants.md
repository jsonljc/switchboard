# D5 — Governance & Architecture Invariants (Riley capability audit, 2026-06-10)

## Thesis

Riley's governed spine is real and verifiably shipped: every mutating Riley path enters `PlatformIngress.submit()`, governance evaluates exactly once, idempotency is claim-first, approval is lifecycle state ending in dispatch-or-recovery, and the baseline's critical Phase-2 prerequisite (no `system_auto_approved` for financial intents) was honored in the Phase-C design. **There is no active bypass or phantom-success path on Riley's surface today.** The gaps are structural, not active: the human gate between Riley and a real Meta write is a single deletable per-org Policy row (intent-level `approvalPolicy` is decorative by admission), the production provisioning path never seeds that row at all, and two observability seams (parked-replay response shape; discarded handoff submit response) corrupt park-truth and alarm fidelity exactly where the system most needs honest signals.

**Verdict: sound-with-gaps.** Before Spec-1B widens the act-leg beyond pause, the platform needs a non-decorative intent-level approval floor (or a last-mile approved-lifecycle check in executors) so that human-gating does not depend on the continued existence of seedable, deletable data rows.

## Findings detail

### P1

**D5-1 (known-open; pilot-spine F-16 territory)** — `seedRileyAdOptimizerDeployment` and `seedMiraCreativeDeployment` are the only writers of the allow + mandatory-approval policy pairs, and they run only from `packages/db/prisma/seed.ts:620` (org_dev). The production deploy route (`marketplace.ts:208-241`) creates the Riley deployment with no policies. Engine default-denies unmatched workflow intents (`policy-engine.ts:589`). Net: for every marketplace-provisioned org, pause is fail-closed-loud, handoff is fail-closed-**silent** (D5-4). Fix: seed at deploy time + a flip-tool preflight.

**D5-2 (net-new)** — The human gate over a real Meta pause is exactly one Policy row. The registration's `approvalPolicy:"always"` is decorative by its own comment (`contained-workflows.ts:430`). The dangerous decomposition is acknowledged and pinned (`riley-pause-gate.test.ts:164` "allow alone EXECUTES") yet remains reachable two ways: (a) the seed's two upserts are not transactional (`seed-riley-ad-optimizer-deployment.ts:83-94`); (b) `DELETE /api/policies/:id` (`policies.ts:164-200`) deletes the approval half org-scoped. The executor performs no approved-lifecycle check; the only detection is the submitter's post-Meta-write error log (`riley-pause-submitter.ts:76-81`). Fix: a non-decorative intent-level mandatory floor, or an executor pre-flight on `trace.approvalOutcome === "approved"`, plus transactional seeding and protected policy ids.

### P2

**D5-3 (net-new)** — `platform-ingress.ts:150-177`: idempotent replays of a `pending_approval` trace return `{ok:true,...}` WITHOUT `approvalRequired`. The pause submitter's branch order then logs `UNEXPECTEDLY executed without approval — investigate governance seeding` and returns `parked:false`. On an Inngest retry replaying the deterministic key this (i) cries wolf on the single log line whose credibility D5-2 depends on, and (ii) corrupts strict-truth `riley_self` ownership. Fix in the ingress replay path (reconstruct `approvalRequired:true` for `pending_approval`).

**D5-4 (net-new)** — `inngest.ts:357-360`: the handoff submitter ignores the `SubmitWorkResponse`. Entitlement blocks, deployment failures, governance denies (guaranteed under D5-1) and unexpected-executes are all invisible. Retrofit the pause submitter's named-branch contract.

**D5-5 (net-new)** — `marketplace.ts:41,236`: `governanceSettings: z.record(z.unknown())` passthrough lets a deploy request set `pauseSelfExecutionEnabled` / `trustLevelOverride` / `spendAutonomy` with no audit row, defeating the chain-hashed capability-assignment contract of `riley-pause-flag-toggle.ts`. Allowlist the keys.

**D5-6 (net-new)** — `internal-ingress.ts:66` + `validation.ts:221-227`: the F-15 hop accepts `trigger:"internal"` and arbitrary actors (incl. seeded `system`), so the "internal-trigger-only, unreachable from the public API" containment on Riley's intents reduces to INTERNAL_API_SECRET possession. Clamp trigger/actor or allowlist intents on the hop.

### Refinements

- **D5-7** — flag flip mutates before the ledger write (`riley-pause-flag-toggle.ts:57-80`); a ledger failure = silent capability change. Ledger-first or transactional.
- **D5-8** — engine trusted-behavior fast path (`policy-engine.ts:581-586`) overrides `policyApprovalOverride === "mandatory"`. Inert (system identity `trustBehaviors: []`, `seed.ts:122`) but one data row away from defeating the pause park. Make mandatory survive trust; pin with a test.
- **D5-9** — `recommendations.ts:183-205` lacks the `approvalRequired` branch; latent 500-on-park if `recommendation.act` ever leaves `system_auto_approved`.

## What is sound (with evidence)

1. **The Phase-C submit safety contract is the best-engineered seam in the audit.** The submitter's branch order is documented as load-bearing, pinned by a dedicated test, and handles the phantom-success gotcha, entitlement-named-skip, governance-deny-loud, and relaxed-gate-loudest cases explicitly (`riley-pause-submitter.ts:26-88`; `riley-pause-submitter.test.ts`).
2. **The real-engine gate test suite is exemplary**: real policy seeds + real engine, pinning park-at-mandatory, autonomous-override-does-not-relax, both decomposition halves, default-deny on unseeded orgs, bespoke-actor hard-deny, and regex anchoring non-bleed (`riley-pause-gate.test.ts:142-198`). E2E loop tests cover flag-off default, flag-on park+approve+Meta-write, primary-only, and entitlement skip (`riley-pause-cron-loop.test.ts`).
3. **Execution truth at the last mile**: Zod fail-closed parse, defense-in-depth floors, 48h staleness cap, org-isolated credential resolution, fresh-client rate-limiter workaround, pre-read abstentions, and the #946/#949 bookkeeping discipline (Meta-write truth never converted into a false `failed`; `not_found` warn, linkage-failure error) — `riley-pause-execution-workflow.ts:109-287`.
4. **Claim-first idempotency (D1)** is fully implemented with fail-closed unresolved-claim and lost-race semantics (`platform-ingress.ts:112-179,376-417`).
5. **Approve-ends-in-dispatch-or-recovery** holds with per-attempt dispatch locks and version-checked retry admission (`lifecycle-dispatch.ts`, `respond-to-parked-lifecycle.ts`).
6. **Baseline 7.1 (the audit's CRITICAL execution prerequisite) is verifiably closed** — the act-leg shipped on `approvalMode:"policy"` + seeded mandatory parks, not the `system_auto_approved` shortcut the baseline warned against.
