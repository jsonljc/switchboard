# D4 — v3 Control Plane & Pause Execution Path

**Auditor:** D4-control-plane | **Date:** 2026-06-10 | **Baseline:** docs/audits/2026-06-02-riley-improvement-audit (8 days stale; pause spine shipped after it: #927/#931/#933/#923/#934/#939/#946/#948/#949)

## Thesis

The Phase-C pause path is the first place Switchboard lets Riley _initiate_ a live ad-platform mutation, and it was built the way the doctrine says revenue actions must be built: candidate decision pure in Layer 2 (`riley-pause-dispatch.ts`), submit through PlatformIngress with the seeded system principal and a deterministic idempotency key, mandatory-approval park enforced by real org-scoped policies (not route logic), execution behind a defense-in-depth workflow handler, failure landing in `recovery_required`, and execution truth flowing back into attribution. I traced the full dark path as if both flags were flipped — weekly audit → arbitration primary → candidate gate → submit-and-park → human approve → dispatch → MetaAdsClient → recommendation transition → attribution candidacy — and found **no P0**: no bypass, no phantom success on the money leg, no double-execution window. The verdict is _sound-with-gaps_: every gap found is at a seam (replay semantics, infra-failure recovery, ambiguous-write attribution, production seeding), not in the spine.

## Findings detail

### D4-1 (P2, net-new) — Replay of a parked submit fires the loudest false alarm

Ingress's cached-replay branch (platform-ingress.ts:119-178) returns `{ok:true, result:{outcome:"pending_approval"}}` **without** `approvalRequired`. The submitter (riley-pause-submitter.ts:63-81) checks approvalRequired, then `outcome==="failed"`, then falls into `"riley pause UNEXPECTEDLY executed without approval … investigate governance seeding"` — the alarm reserved for a relaxed gate. The weekly cron has `retries: 2` and recommendation emission is day-bucket idempotent (emit.ts:15-23), so a same-day step retry deterministically replays the same submit key and fires it. Side effect: `parked:false` → `pauseParkedIndex` undefined → the retry run's report drops `riley_self` for a pause that is genuinely parked. The approval-loop test pins the ingress shape (test:139-153) but nothing tests the submitter consuming it. **Fix (S):** treat `result.outcome === "pending_approval"` as park truth.

### D4-2 (P2, net-new, partially platform-domain) — Approved-but-undispatched lifecycles have no retry surface

`recovery_required` is only set after `prepareDispatch` succeeds. A throw in `writeApprovedPayloadToTrace` (integrity-locked trace), a `prepareDispatch` failure, or process death before `recordDispatchOutcome` leaves the lifecycle `approved`; respond-to-parked-lifecycle.ts:115-117 then rejects every further respond as `already_responded`. A human-approved spend mutation can strand with no operator-facing recovery. **Fix (M):** allow approve-on-approved-with-no-terminal-dispatch to re-enter `retryDispatch`, or sweep approved-undispatched lifecycles into `recovery_required`.

### D4-3 (P2, net-new) — Ambiguous Meta write makes Riley's own pause invisible to attribution

Timeout-but-applied write → `failed` → recovery retry → pre-read sees PAUSED → abstains `campaign_already_paused`, which is _deliberately excluded_ from the acted transition (execution-workflow.ts:66-70,185-198, pinned by test:363). In this leg the exclusion's rationale ("someone else's action") is wrong — it was our write. The rec dies pending; the executed pause never reaches the outcome ledger. **Fix (M):** on retry, consult prior DispatchRecords to distinguish own-ambiguous-write from external pause.

### D4-4 (P2, net-new) — No reconciliation for "Meta paused but linkage failed"

The transition-error leg records `recommendationTransition:"error"` in outputs and console.errors, then completes (execution-workflow.ts:253-263). Zero consumers read that output; nothing retries the (idempotent) transition. **Fix (M):** reconciliation pass over completed pause traces with `recommendationTransition==="error"`.

### D4-5 (P1, known-open — F-16 family) — No production policy-seeding path; flip tool doesn't verify

`seedRileyAdOptimizerDeployment` (which upserts both pause policies) runs only for `org_dev` (prisma/seed.ts:620). `scripts/riley-pause-flag.ts` checks the deployment, not the policies. Unseeded + flag-on = default-deny on every weekly submit (fail-closed, loud log — gate test:184) but the feature is permanently inert and there is no flip runbook under docs/runbooks/. **Fix (S):** `--enable` asserts/upserts both policies via the shared builders; write the flip runbook.

### D4-6 (P2/refinement, net-new) — Floor bounds evidence, not blast radius; guardrails declarative

No spend-share cap (Riley can park a pause on the org's largest campaign), no cross-run cooldown, and `PHASE_C_EXECUTION_SEAM.pause.guardrailMetrics` are recorded into outputs and monitored by nothing (the attribution cron lags windowDays+48h). Fine while a human is the bound; **disqualifying as the Spec-1B autonomy template**. Needs: spend-share gate, daily post-pause guardrail monitor, governed resume intent (today rollback is Ads Manager by hand — the client throws on ACTIVE by design).

### D4-7 (refinement) — Rec card and parked approval card are unlinked levers for the same pause

Dismissing the rec doesn't withdraw the park; rejecting the park doesn't dismiss the rec. Row integrity is safe (status-predicate serialization, benign not_pending), but a pause can execute after its recommendation was explicitly dismissed.

### D4-8 (refinement) — Unvalidated credential casts produce "Bearer undefined" failures

riley-pause-executor.ts:84-87 casts accessToken/accountId `as string`; missing fields surface as META_PAUSE_FAILED instead of a named misconfiguration.

## What is sound (with evidence)

- **The governance posture is real and pinned, not asserted.** The gate tests run the actual engine against the same policy builders production seeds: mandatory park (test:143), autonomous-override no-relax (test:152), allow-alone-executes decomposition hazard (test:164), unseeded default-deny (test:184), bespoke-system-principal hard-deny (test:190), anchored regex no-bleed (test:197). The intent registration carries no `system_auto_approved` (contained-workflows.ts:422-438) — the #788 landmine was explicitly designed around.
- **Idempotency holds at every layer.** Recommendation emission dedupes by org+intent+target+day (emit.ts:15-23); the pause submit key is deterministic per recommendation; ingress returns the cached park on replay with no second lifecycle (approval-loop.test.ts:139); dispatch is locked per attempt (`lifecycle-dispatch:<id>:<rev>:attempt-<n>`); the executor's pre-read makes even a cross-park double-approve converge (second abstains `campaign_already_paused`); markActedByExecution is first-writer-wins on the status predicate.
- **Approve-ends-in-dispatch-or-recovery holds for the dominant failure mode.** Meta write failure → `failed` → DispatchRecord failed → `recovery_required` → approve-on-recovery IS retry with strict re-admission (respond-to-parked-lifecycle.ts:207-244), proven end-to-end (approval-loop.test.ts:109, executed-attribution-loop.test.ts:99 — retry transitions exactly once).
- **Execution truth is recorded, not implied.** previousStatus/newStatus/metaWriteAccepted/ageHours/recommendationTransition/executedAt all land in outputs; the stale cap (48h) sits honestly _behind_ the platform's 24h park expiry and is documented as a backstop, with the boundary tested on both sides (workflow test:152,173,277).
- **The strict-truth ownership discipline survived the wiring.** `riley_self` comes only from the sink's park fact; a flag-on-but-failed submit stays `operator_approval` (recommendation-ownership.ts:100-106) — exactly the anti-illusion rule the autonomy-fields gotcha demands.

## Readiness verdict (Spec-1B act-leg flip)

**Flip the two flags for a pilot org: YES, after two preconditions.** (1) Seed the org's pause policies and make the flip tool verify them (D4-5); (2) fix the replay false alarm so the one real safety alarm stays trustworthy (D4-1). The human-approved seam itself is the best-engineered execution path in the Riley surface — tested end-to-end against the real governance and dispatch stack.

**Use this seam as the Spec-1B autonomy template as-is: NO.** Removing the human removes the only blast-radius bound that is actually enforced. Before any autonomous arm: spend-share cap at the candidate gate, real-time (daily) guardrail monitoring wired to the declared guardrailMetrics, a governed resume/rollback intent, and resolution of D4-2/D4-3 so infra failures cannot strand or mis-attribute an autonomous money action.
