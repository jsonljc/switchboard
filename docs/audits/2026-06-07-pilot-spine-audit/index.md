**Method:** Architecture audit + Route-chain audit. Every finding follows the evidence
standard (exact file path, exact function/route/component, observed behavior, expected
behavior, customer/product impact, recommended fix, validation/test). Each route-chain
step is traced end-to-end: frontend → hook/client → dashboard proxy → backend route →
service/store → database/external provider. Status per step is one of
PASS / FAIL / STUB / NO-OP / MISSING.

Exercised + reproduced (J1-S3).
- [F-16](findings/F-16-fresh-org-no-require-approval-policy-nothing-parks.md) — a fresh self-serve pilot org has ZERO per-org Policy/IdentitySpec rows (they are seeded for `org_dev` only; per-org provisioning is a seed TODO, `creative-governance.ts:21`). Two-sided impact: (A) operator-reachable actions have no `require_approval` policy to match ⇒ approval Inbox permanently empty; (B) agent-initiated governed actions hit default-deny floors (empty Policy set + missing IdentitySpec) ⇒ agents cannot take any governed actions. Positive: this is the safe direction (no default-allow / unapproved-exec exposure for fresh customers). The respond→dispatch lifecycle is correctly written but its producer never fires. Confirmed live (0 Policy/0 ApprovalLifecycle/0 WorkTrace; empty decisions feed). Corrects the J3-S5 handoff assumption that the policies were seeded for this org.
- [F-15](findings/F-15-chat-api-ingress-unauthenticated-at-prod-default.md) — (by-config, all-channel, launch-checklist) `SWITCHBOARD_API_KEY` is absent from `render.yaml` for the `switchboard-chat` service and blank in `.env.example`; the adapter at `http-platform-ingress-adapter.ts:19-21` is correctly written (sends `Authorization: Bearer <key>` when the key is set), but without a provisioned key the chat server sends no auth header and `/api/ingress/submit` 401s on every managed-channel message — WhatsApp and Telegram alike. Fix is provisioning the key (+ matching `API_KEYS`/`DashboardUser.apiKeyHash` on the API side); this is a deploy-config gap in the same family as F-03, not an adapter code bug. Exercised + reproduced live on Telegram path (J3-S3); all-channel by nature.
